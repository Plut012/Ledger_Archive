## How to Use These Documents

### Starting a Phase

1. **Read the context document**
   - Understand what you inherit
   - See what you need to build
   - Review success criteria

2. **Read the detailed plan**
   - Get full implementation details
   - Understand step-by-step approach

3. **Start coding**
   - Follow the plan
   - Refer to context as needed

### Ending a Phase

1. **Fill out the handoff document**
   - Check all checkboxes
   - Document implementation details
   - Note issues/limitations
   - Add helpful notes

2. **Verify completion**
   - Run all tests
   - Test user flows
   - Check success criteria

3. **Update START_HERE.md**
   - Point to next phase
   - Update current status

4. **Ready for next phase**
   - Next context document is ready
   - Clear handoff completed

## Document Structure

### Context Documents
```markdown
- What you inherit (previous phases)
- Your goal
- What to build
- Implementation plan
- Files to modify
- Success criteria
- Code templates
- Testing guide
```

### Handoff Documents
```markdown
- What you inherited
- What you built
- Implementation details (fill in)
- Features implemented (checklist)
- Known issues (document)
- Completion checklist (verify)
- Notes for next phase (helpful tips)
```
